## Step-by-Step Organization

### 1. Run the Organization Script

```bash
cd /path/to/resume-backend
chmod +x organize-scripts.sh
./organize-scripts.sh
```

This will:
- Create `scripts/gauge-setup/` directory
- Move all scripts there
- Create `docs/` directory  
- Move all documentation there
- Make scripts executable
- Create README in scripts directory

### 2. Update .gitignore

Add these lines to your `.gitignore`:

```bash
cat gitignore-additions.txt >> .gitignore
```

Or manually add:
```
# Gauge
libs/gauge/
logs/
reports/
env/default/*.backup
```

### 3. Update Project README

Add a section to your main `README.md`:

```markdown
## Gauge BDD Tests

### Setup

For first-time setup or troubleshooting:

\`\`\`bash
./scripts/gauge-setup/setup-gauge.sh
\`\`\`

### Running Tests

\`\`\`bash
# All tests
./gradlew gaugeTest

# Specific spec
./gradlew gaugeRun -Pspec=specs/profile_api.spec

# Direct (after setup)
gauge run specs
\`\`\`

### Troubleshooting

If tests fail, run diagnostics:

\`\`\`bash
./scripts/gauge-setup/diagnose-gauge.sh
\`\`\`

See [docs/GAUGE_TROUBLESHOOTING.md](docs/GAUGE_TROUBLESHOOTING.md) for detailed help.
```

### 4. Commit Everything

```bash
# Stage the new structure
git add scripts/ docs/ env/default/java.properties build.gradle

# Update .gitignore
git add .gitignore

# Commit
git commit -m "Add Gauge BDD test setup with troubleshooting scripts

- Add Gauge configuration and dependency management
- Add diagnostic and setup scripts
- Add comprehensive troubleshooting documentation
- Fix SLF4J conflict with Spring Boot 3.x
- Update build.gradle with Gauge tasks
- All 9 BDD scenarios passing"
```

## What to Keep vs. Delete

### ‚úÖ Keep (Commit to Git)

**Scripts** - Valuable for team & CI/CD:
- All scripts in `scripts/gauge-setup/`
- README in scripts directory

**Documentation** - Reference material:
- All markdown files in `docs/`
- Shows solutions to real problems
- Helps new developers

**Configuration**:
- `env/default/java.properties` - Critical for Gauge
- Updated `build.gradle` - Gradle tasks for Gauge
- `.gitignore` additions - Prevents committing generated files

### ‚ùå Don't Commit (Generated Files)

**Generated Dependencies**:
- `libs/gauge/*.jar` - Regenerated by `./gradlew copyGaugeDependencies`
- These are ~150 JAR files, ~100MB

**Build Outputs**:
- `build/` - Gradle build directory
- `logs/` - Gauge logs
- `reports/` - Test reports

**Backups**:
- `*.backup` files created by scripts
- Temporary diagnostic files

## Why Keep the Scripts?

### 1. Team Onboarding
New developers can run:
```bash
./scripts/gauge-setup/setup-gauge.sh
```

And be ready to work immediately.

### 2. CI/CD Integration
Your CI pipeline can use:
```bash
./scripts/gauge-setup/setup-gauge.sh
./gradlew clean build test gaugeTest
```

### 3. Troubleshooting
When something breaks (and it will), the scripts show:
- How you fixed it before
- What the issue was
- How to diagnose similar problems

### 4. Documentation
The scripts ARE documentation:
- They show working solutions
- They're runnable examples
- They're version controlled
